The main product is GrowBot-with-wheels: a curious, LLM-brained companion robot — an onboard fast loop (drive + reflexes) plus a cloud brain that remembers, dreams, expresses itself, and talks. Playing with Roy is one behaviour it can choose, not the whole point. ~$100, and fully onboard except the cloud brain + nightly dreams (vision runs on the Pi, like GrowBot's face-tracking — no PC offload).

Phase 1

Roll the GrowBot foundation

NOW Buy ~$99

Goal: a physical bot that drives, senses, and tracks Roy onboard — the GrowBot body, wheeled.

Key work

  • Order the ~$99 BOM (you already have the Pi + 3216 kit + filament)
  • Assemble: solder header; print the motor adapter, cover/shroud, mounts; wire the dual rail (5.1 V Pi / 6.0 V motors), generic N20+encoders, IMU, camera, mic/amp/speaker, LED
  • Driver bring-up: closed-form drive + an encoder PID that closes the wheel-speed loop (matches the sim)
  • Onboard 50 Hz runtime; onboard visual tracking — Roy's white-chest blob + general awareness (no PC)

Risks & mitigations

  • Stall brownout → dual rail + bulk cap + PWM soft-start + drive watchdog
  • Bring-up surprises → stage it (solder → flash → meter-set rails → bench driver self-check → tethered drive)

Success

  • Drives + turns in place; all sensors live; tracks Roy's white chest onboard; IMU tip/stall reflexes fire
Phase 2

The brain (LLM)

NEXT the main product's mind

Goal: the cloud LLM brain — the heart of the GrowBot idea. Strictly out of the 50 Hz loop; cloud is for language/reflection only.

Key work

  • Swappable OpenRouter client (deepseek/deepseek-v4-flash to start; model id from config; key from env + .gitignore + hard spend cap)
  • A memory it reads/writes — the agent loop (a model in a loop with memory)
  • The LLM drives the body by selecting/tuning the fast policies and writing small code snippets on the fly (like GrowBot)
  • Emotional expression — coordinated motion + sound + LED states
  • Two-way voice — wake-word + STT + TTS (prefer local STT/TTS: lower latency, keeps raw audio off the cloud)

Risks & mitigations

  • Key safety → env var + .gitignore + spend cap before the first commit that touches it
  • Cloud/WiFi dependency → degrade to a no-op offline; never block the robot on a slow call
  • Privacy → send only abstracted text to the cloud, never raw audio/images

Success

  • Responds to touch/voice, expresses believable states, composes actions via the LLM, and talks with you
Phase 3

Memory + dreams

LATER home base / cloud

Goal: it learns over time and consolidates each night — GrowBot's "dream."

Key work

  • Persistent memory — profiles of Roy, you, and events
  • Nightly dream: dump the whole memory to the smartest model to consolidate, resolve conflicts, and extract lessons; write the daily Roy report
  • Optionally fine-tune the chase policy on real interactions

Success

  • Memory builds; the dream runs nightly; a daily report lands; behaviour measurably improves over days
Phase 4

Cat-play behaviour

LATER ready to slot in early

Goal: wire the already-trained chase policy in as a behaviour the brain invokes, on her terms.

Key work

  • Deploy the chase policy; feed it the onboard white-chest CatState (bearing / range / approach / engagement / visible)
  • The brain's arbiter decides play vs leave-alone vs explore; consent — back off when Roy is done
  • Obs/reward reframe (blob-tracker signals + heuristic engagement + a trained cat-lost path) → retrain

Note: the policy already exists, so this is technically ready right after Phase 1 — pull it earlier if you want a quick "it chases Roy!" win before the full brain.

Success

  • Plays with Roy on her terms, fully onboard, invoked by the brain
