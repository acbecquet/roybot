Phase 1

First prototype build

NOW BOM Stage A (~$158)

Goal: a physical bot that drives, runs the policy, and visibly reacts to Roy (perception offloaded to the PC).

Key work

  • Order the Stage-A BOM (~$158)
  • Assemble: solder header; print + mount the N20→3216 adapter, cover/shroud, lure boom; wire the dual rail (meter-set 5.1 V / 6.0 V), TB6612, encoders, IMU, camera, audio, LED
  • Driver bring-up: self-check (straight = equal, spin = opposite, forward-polarity); encoder PID closes the wheel-speed loop (matches the sim)
  • Deploy the policy: NumPy MLP @50 Hz on the Pi, frame-stack N_STACK=6; verify obs order/scale vs env.py
  • Perception Stage A: stream camera → PC detector → CatState back over WiFi
  • Prove "reacts to Roy" with a hand-written turn-toward-blob controller before swapping in the MLP

Risks & mitigations

  • Power brownout on stall → dual rail + bulk cap + PWM soft-start + drive watchdog
  • Cat-lost on dropout → handle visible=False (safe idle), never feed stale obs
  • Obs/reward entanglement → addressed by the Phase 2 reframe (hand-written controller bridges until then)

Success criteria

  • Drives forward + turns in place; executes policy velocity commands
  • Safety floor cuts motors on stall/tip; visibly tracks + reacts to Roy

Phase 2

Standalone perception

NEXT Perception Stage B · +collar ~$5

Goal: cut the PC cord — perception runs entirely onboard the Pi Zero.

Key work

  • Onboard colored-collar HSV blob tracker (5–15 Hz) + hold/constant-velocity estimator → CatState; collar on Roy
  • Obs/reward reframe (the reality-check fix): retrain on bearing / range / approach-sign / engagement / visible; add a trained cat-lost path; widen latency DR to 250–500 ms; down-sample cat obs to 5–10 Hz
  • Calibrate blob-area → range against the collar's known size

Why it works onboard

  • Blob tracking is light enough for the Zero 2 W (a cat neural net is not — that's Stage C)
  • Same bot as Phase 1; only addition is the collar

Success criteria

  • Plays with Roy with the PC off and WiFi irrelevant

Phase 3

Memory + the cloud brain

LATER Sub-projects #3 / #4

Goal: Roybot remembers Roy and gains a language/reflection brain (out of the fast play loop).

Key work

  • Interaction logging + preference learning (#3): learn Roy's likes/dislikes + schedule, feed the behavior arbiter
  • Swappable LLM client: OpenRouter deepseek/deepseek-v4-flash, async, model id from config (deepseek-v4-pro one-line fallback)
  • Home base (PC): store recordings; nightly "dream" — consolidate logs, update Roy-prefs, optionally fine-tune the policy

Risks & mitigations

  • Key handling → OPENROUTER_API_KEY from env + .gitignore + hard spend cap, before the first commit
  • Cloud needs WiFi → degrade to a no-op offline; never block the robot on a slow call
  • Privacy → send only abstracted text to the cloud, never raw audio/images

Success criteria

  • Daily Roy report generated; preferences measurably shift behavior; dream runs nightly

Phase 5

Self-charging dock

LATER Sub-project #6 · dock parts

Goal: Roybot returns and charges itself, and docking triggers the nightly dream.

Hardware

Success criteria

  • Bot docks + charges unattended; dream fires on dock
